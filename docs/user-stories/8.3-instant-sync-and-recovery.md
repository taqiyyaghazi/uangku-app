# Story 8.3: Instant Sync & Recovery (Data Restoration)

**Status:** Ready for Implementation
**Epic:** 8 - Cloud Synchronization
**Story Point:** 5 (Logic & State Management)

---

## User Story

**As a** user,
**I want** my data to be automatically restored when I log in on a new device,
**so that** I don't lose my financial history and can continue tracking seamlessly.

---

## Acceptance Criteria

**AC #1: Initial Data Pull (Hydration)**

- **Given** user baru saja login di perangkat baru (Database lokal kosong),
- **When** masuk ke Dashboard untuk pertama kalinya,
- **Then** lakukan penarikan data massal (_Batch Fetch_) dari Firestore untuk koleksi `categories`, `wallets`, dan `transactions`.

**AC #2: Local Storage Persistence**

- **Given** data dari Firestore,
- **When** diterima oleh aplikasi,
- **Then** simpan data tersebut ke dalam database **Drift (SQLite)** lokal agar aplikasi tetap bisa bekerja secara offline di kemudian hari.

**AC #3: Duplicate Prevention (Upsert)**

- **Given** proses sinkronisasi,
- **When** menyimpan data ke Drift,
- **Then** gunakan logika **"Insert or Replace"** (Upsert) berdasarkan ID unik transaksi, agar tidak terjadi data ganda jika user melakukan _Clear Data_ atau _Re-login_.

**AC #4: Sync Progress Feedback**

- **Given** proses penarikan data besar,
- **When** sedang berlangsung,
- **Then** tampilkan _Overlay Loading_ atau _Progress Bar_ bertuliskan _"Restoring your data from cloud..."_ agar user tidak mengira aplikasi _hang_.

---

## Implementation Details (Winston's Technical Guide)

### 📦 Logic Flow

1. **Check Local State:** Cek apakah tabel `Categories` sudah ada isinya.
2. **Fetch Cloud:** Jika kosong, panggil `FirebaseFirestore.instance.collection('users').doc(uid).collection('transactions').get()`.
3. **Map & Save:** Ubah `QueryDocumentSnapshot` menjadi objek Drift, lalu jalankan `batch` insert:

```dart
await batch((b) {
  for (var row in cloudData) {
    b.insertAll(table, [row], mode: InsertMode.insertOrReplace);
  }
});

```

### Tasks / Subtasks

- [ ] **Sync Service:**
- Buat fungsi `syncFromCloud()` yang dipanggil tepat setelah Login sukses.

- [ ] **Dependency Ordering:**
- Penting! Restore **Categories** dan **Wallets** dulu sebelum **Transactions** agar tidak terjadi error _Foreign Key Constraint_.

- [ ] **Timestamp Logic:**
- Gunakan field `updated_at` untuk hanya menarik data yang lebih baru dari data lokal (untuk penghematan kuota Firebase).

### Technical Summary

Tantangan di sini adalah **Data Integrity**. Karena Firestore tidak punya _Foreign Key_, Anda harus memastikan ID kategori di Cloud benar-benar ada di tabel kategori lokal sebelum transaksi dimasukkan. Jika tidak, database SQLite (Drift) Anda akan protes (Crash).
