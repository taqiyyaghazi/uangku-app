# Story 7.1: Edit Transaction Date (Backdating Adjustment)

**Status:** Ready for Implementation
**Epic:** 7 - Data Integrity & Portability
**Story Point:** 2 (Low Complexity, High Impact)

---

## User Story

**As a** user,
**I want** to be able to change the date of an existing transaction,
**so that** I can fix mistakes and keep my financial records accurate.

---

## Acceptance Criteria

**AC #1: Editable Date Field**

- **Given** layar "Edit Transaction" (yang muncul saat transaksi di tap/tahan),
- **When** layar tersebut terbuka,
- **Then** tampilkan field Tanggal (sama seperti di Quick Entry) yang bisa diklik untuk memicu `showDatePicker`.

**AC #2: Date Persistence**

- **Given** user memilih tanggal baru,
- **When** tombol "Update" atau "Save" ditekan,
- **Then** sistem harus mengupdate kolom `date` di tabel `Transactions` sesuai pilihan baru tersebut.

**AC #3: Dashboard Synchronization**

- **Given** perubahan tanggal lintas bulan (misal: dari 1 Maret ke 28 Februari),
- **When** data disimpan,
- **Then** "Daily Breath" untuk bulan Maret dan Februari harus otomatis menghitung ulang sisa saldo masing-masing bulan.

**AC #4: Consistent UI State**

- **Given** modal edit dibuka,
- **When** pertama kali muncul,
- **Then** tanggal yang tertera haruslah tanggal asli transaksi tersebut, bukan tanggal hari ini (`DateTime.now()`).

---

## Implementation Details (Winston's Technical Guide)

### Tasks / Subtasks

- [ ] **UI Update:**
- Tambahkan `Date Selector` widget ke dalam `EditTransactionSheet`.
- Gunakan state lokal `_selectedDate` yang diinisialisasi dari `transaction.date`.

- [ ] **Logic Sync:**
- Pastikan fungsi `updateTransaction` di DAO menerima parameter `date` terbaru.

- [ ] **State Management:**
- Jika menggunakan `Provider` atau `Bloc`, pastikan memicu _refetch_ pada data dashboard agar saldo langsung berubah setelah modal ditutup.

### Technical Summary

Ini adalah perbaikan pada form validation. Pastikan user tidak bisa memilih tanggal di masa depan (sesuai aturan di Story 4.7). Jika aplikasi Anda mendukung jam (Time), pastikan saat edit tanggal, informasi waktu (HH:mm) tetap dipertahankan atau direset secara logis ke jam 12:00 siang agar urutan kronologis tetap terjaga.
