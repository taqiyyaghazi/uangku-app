# Story 7.3: Export Transactions to CSV

**Status:** Ready for Implementation
**Epic:** 7 - Data Integrity & Portability
**Story Point:** 5 (Medium-High: File System & External Sharing)

---

## User Story

**As a** user,
**I want** to export my transaction data to a CSV file,
**so that** I can perform deeper analysis in Excel or keep a physical backup of my records.

---

## Acceptance Criteria

**AC #1: Export Trigger Point**

- **Given** layar Settings atau Insights,
- **When** user menekan tombol "Export to CSV",
- **Then** tampilkan _Loading Indicator_ sementara aplikasi memproses data.

**AC #2: Data Formatting (CSV Mapping)**

- **Given** seluruh data transaksi di database,
- **When** proses ekspor berjalan,
- **Then** ubah data tersebut menjadi format CSV dengan kolom:
  `Date, Amount, Category, Wallet, Type, Notes`
- **Note:** Pastikan nama kategori dan nama wallet yang muncul adalah teks (String), bukan sekadar ID angka.

**AC #3: File Generation & Storage**

- **Given** string CSV yang sudah jadi,
- **When** diproses,
- **Then** simpan file tersebut sementara di direktori dokumen aplikasi (`path_provider`) dengan nama dinamis: `Uangku_Export_YYYYMMDD.csv`.

**AC #4: Share Dialog Integration**

- **Given** file CSV sudah siap di penyimpanan,
- **When** proses selesai,
- **Then** otomatis munculkan sistem "Share Sheet" Android/iOS agar user bisa memilih aplikasi (WhatsApp, Email, Drive) untuk mengirim file tersebut.

---

## Implementation Details (Winston's Technical Guide)

### 📦 Required Libraries

- `csv`: Untuk konversi List ke String CSV.
- `path_provider`: Untuk akses folder penyimpanan.
- `share_plus`: Untuk memicu dialog sharing.

### Tasks / Subtasks

- [ ] **Data Fetching:**
- Buat query `getAllTransactionsWithDetails()` yang melakukan `JOIN` antara tabel `Transactions`, `Categories`, dan `Wallets`.

- [ ] **CSV Logic:**
- Buat fungsi `generateCsv(List<Transaction> data)` yang menyusun header dan baris data.

- [ ] **File & Share Flow:**
- Implementasi logic: `Data -> CSV String -> Write to File -> Share.shareXFiles()`.

### Technical Summary

Tantangan teknis utama adalah **Data Formatting**. Tanggal harus diubah ke format yang bisa dibaca Excel (misal: `yyyy-MM-dd`), dan nominal harus berupa angka murni tanpa simbol mata uang agar bisa langsung dijumlahkan di spreadsheet. Gunakan `try-catch` blok yang kuat untuk menangani izin akses penyimpanan (_storage permissions_).
