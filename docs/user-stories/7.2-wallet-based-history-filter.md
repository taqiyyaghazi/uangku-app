# Story 7.2: Wallet-Based History Filter

**Status:** Ready for Implementation
**Epic:** 7 - Data Integrity & Portability
**Story Point:** 3 (Data Querying Complexity)

---

## User Story

**As a** user,
**I want** to filter my transaction history by a specific wallet,
**so that** I can reconcile the balance of my physical wallet (like Cash or Bank) with the app's records.

---

## Acceptance Criteria

**AC #1: Wallet Filter UI (Chip Bar)**

* **Given** layar History Transaksi,
* **When** layar dibuka,
* **Then** tampilkan baris horizontal berisi pilihan Wallet (Filter Chips) di bawah Header/Search bar.
* **Pilihan pertama** harus selalu "All Wallets".

**AC #2: Dynamic Filtering Logic**

* **Given** daftar transaksi,
* **When** user menekan salah satu chip wallet (misal: "Bank Mandiri"),
* **Then** daftar transaksi harus diperbarui secara instan hanya untuk menampilkan transaksi yang melibatkan wallet tersebut (baik Expense, Income, maupun Transfer In/Out).

**AC #3: Visual Selection State**

* **Given** Filter Chips,
* **When** sebuah wallet dipilih,
* **Then** beri penekanan visual (misal: warna background chip berubah menjadi warna tema aplikasi) agar user tahu filter mana yang sedang aktif.

**AC #4: Persistent Search Integration**

* **Given** filter wallet aktif,
* **When** user juga mengetik di kolom pencarian (Search Bar),
* **Then** hasil yang muncul harus merupakan irisan keduanya (Mencari teks hanya di dalam wallet yang dipilih).

---

## Implementation Details (Winston's Technical Guide)

### Tasks / Subtasks

* [ ] **Data Logic (Drift/SQLite):**
* Perbarui query di DAO: `watchAllTransactions(int? walletId)`.
* Jika `walletId` adalah null, kembalikan semua data. Jika ada, tambahkan klausa `WHERE wallet_id = :walletId OR to_wallet_id = :walletId`.


* [ ] **UI Component:**
* Gunakan `ChoiceChip` atau `FilterChip` di dalam `ListView.builder` horizontal.
* Bungkus daftar transaksi dengan `StreamBuilder` yang memantau perubahan ID wallet yang dipilih.


* [ ] **Empty State:**
* Jika wallet yang dipilih tidak memiliki transaksi, tampilkan ilustrasi/pesan: *"Belum ada transaksi di wallet ini."*



### Technical Summary

Tantangan di sini adalah menangani **Transfer**. Karena transfer melibatkan dua wallet, query Anda harus memastikan bahwa transaksi transfer muncul di kedua wallet tersebut (baik saat melihat pengirim maupun penerima). Gunakan logika `OR` pada kolom `wallet_id` dan `to_wallet_id`.
