# Story 8.2: Database Mapping to Firestore (Cloud Sync)

**Status:** Ready for Implementation
**Epic:** 8 - Cloud Synchronization
**Story Point:** 8 (High: Data Architecture & Sync Logic)

---

## User Story

**As a** user,
**I want** my transactions to be automatically saved to the cloud as soon as I create them,
**so that** I don't lose my data even if I uninstall the app.

---

## Acceptance Criteria

**AC #1: User-Centric Data Structure**

- **Given** user yang sudah terautentikasi (Logged In),
- **When** data disinkronkan,
- **Then** simpan data di Firestore dengan struktur koleksi:
  `users/{userId}/transactions/{transactionId}`
  `users/{userId}/categories/{categoryId}`
  `users/{userId}/wallets/{walletId}`

**AC #2: Automated Upload (Real-time Sync)**

- **Given** transaksi baru (Create), edit (Update), atau hapus (Delete) di lokal,
- **When** aksi tersebut sukses di Drift (SQLite),
- **Then** kirim payload data yang sama ke Firestore secara asinkron.

**AC #3: Offline Graceful Handling**

- **Given** kondisi HP tidak ada internet,
- **When** user melakukan transaksi,
- **Then** aplikasi tidak boleh _error_ atau _freeze_. Gunakan fitur _Offline Persistence_ bawaan Firestore agar data terantre dan terkirim otomatis saat internet kembali.

**AC #4: Cloud Data Integrity**

- **Given** data di Firestore,
- **Then** pastikan field `updated_at` (Timestamp) selalu diperbarui agar sistem tahu versi data mana yang paling baru jika terjadi konflik.

## Implementation Details (Winston's Technical Guide)

### 📦 Required Libraries

- `cloud_firestore`: Database NoSQL dari Firebase.

### Tasks / Subtasks

- [ ] **Firestore Security Rules:**
- Atur _Security Rules_ agar user hanya bisa membaca/menulis data milik mereka sendiri berdasarkan `request.auth.uid`.

- [ ] **Data Repository Refactoring:**
- Buat `SyncRepository` yang membungkus `DriftDatabase` dan `FirebaseFirestore`.
- Setiap fungsi `insert/update/delete` di repository harus memanggil kedua layanan tersebut.

- [ ] **Data Mapping:**
- Buat fungsi konversi dari **Drift Object** (Data Class) ke **Map<String, dynamic>** untuk Firestore, dan sebaliknya.

### Technical Summary

Ingat, Firestore adalah NoSQL (Document-based), sedangkan Drift adalah Relational (Table-based). Anda harus menyimpan `categoryId` dan `walletId` sebagai string/referensi di dalam dokumen transaksi agar relasi datanya tetap terjaga saat di-restore nanti.
