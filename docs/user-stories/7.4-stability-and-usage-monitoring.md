# Story 7.4: Stability & Usage Monitoring (Firebase SDK)

**Status:** Ready for Implementation
**Epic:** 7 - Data Integrity & Portability
**Story Point:** 3 (Configuration Complexity)

---

## User Story

**As a** developer,
**I want** to receive automatic crash reports and track feature usage,
**so that** I can proactively fix bugs and understand which features are most valuable to my users.

---

## Acceptance Criteria

**AC #1: Real-time Crash Reporting**

- **Given** aplikasi mengalami _unhandled exception_ atau _native crash_,
- **When** aplikasi dijalankan kembali,
- **Then** kirim laporan lengkap (stack trace, device info) ke Firebase Crashlytics Console.

**AC #2: Custom Error Logging**

- **Given** kegagalan yang tidak sampai membuat aplikasi mati (misal: gagal simpan file CSV),
- **When** error ditangkap (`try-catch`),
- **Then** kirim _Non-Fatal Error_ ke Crashlytics agar developer tahu ada masalah di logika bisnis.

**AC #3: Feature Engagement Tracking**

- **Given** user menggunakan fitur baru di Epic 7,
- **When** aksi dilakukan (Edit, Filter, Export),
- **Then** kirim _Custom Event_ ke Google Analytics (misal: `event_name: "export_csv_success"`).

**AC #4: Automated Screen Tracking**

- **Given** user berpindah layar (Dashboard ➡️ History ➡️ Insights),
- **When** layar berubah,
- **Then** Google Analytics mencatat nama layar yang sedang diakses secara otomatis.

## Implementation Details (Winston's Technical Guide)

### 1. Dependency Setup (`pubspec.yaml`)

```yaml
dependencies:
  firebase_core: latest_version
  firebase_analytics: latest_version
  firebase_crashlytics: latest_version
```

### 2. Android Configuration (`build.gradle.kts`)

Karena Anda menggunakan Kotlin DSL, tambahkan plugin Crashlytics di `android/build.gradle.kts` (root) dan `android/app/build.gradle.kts`.

**Di Root `build.gradle.kts`:**

```kotlin
plugins {
    // ... plugin lainnya
    id("com.google.firebase.crashlytics") version "2.9.9" apply false
}

```

**Di App `build.gradle.kts`:**

```kotlin
plugins {
    // ... plugin lainnya
    id("com.google.firebase.crashlytics")
}

```

### 3. Initialize in `main.dart`

Bungkus `runApp` untuk menangkap error global:

```dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Firebase.initializeApp();

  // Pass all uncaught errors to Crashlytics
  FlutterError.onError = FirebaseCrashlytics.instance.recordFlutterFatalError;

  // Pass errors outside of Flutter context
  PlatformDispatcher.instance.onError = (error, stack) {
    FirebaseCrashlytics.instance.recordError(error, stack, fatal: true);
    return true;
  };

  runApp(const MyApp());
}

```

## Technical Summary

Hati-hati dengan **Debug vs Release mode**. Secara default, Crashlytics sering kali dinonaktifkan di mode Debug agar tidak mengotori data asli. Anda bisa memaksa aktif untuk testing dengan `await FirebaseCrashlytics.instance.setCrashlyticsCollectionEnabled(true);`.
