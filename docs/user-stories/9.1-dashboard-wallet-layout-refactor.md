# Story 9.1: Dashboard Wallet Layout Refactor (Carousel View)

**Status:** Ready for Implementation
**Epic:** 9 - Power User Experience (UX Refactor)
**Story Point:** 3 (UI Polish & Layout Logic)

---

## User Story

**As a** power user with many wallets,
**I want** to see my wallet balances in a compact horizontal carousel,
**so that** I can see more of my Recent Transactions on the dashboard without scrolling too far.

---

## Acceptance Criteria

**AC #1: Compact Horizontal Carousel**

* **Given** Dashboard utama,
* **When** memiliki lebih dari 2 wallet,
* **Then** ganti tampilan *Grid 2 Kolom* menjadi *Horizontal Scrollable List* (Carousel).
* **Size:** Tinggi area wallet harus tetap (misal: `120px - 150px`) agar konsisten.

**AC #2: Balanced Information Display**

* **Given** sebuah kartu wallet dalam carousel,
* **Then** tampilkan info minimalis: Nama Wallet, Icon, dan Saldo.
* **UX:** Kartu harus cukup lebar agar saldo besar tidak terpotong (Truncated).

**AC #3: Better "Recent Transactions" Visibility**

* **Given** layout baru,
* **When** dashboard dibuka,
* **Then** setidaknya 3-5 baris transaksi terbaru harus terlihat di layar pertama (Above the fold) tanpa user perlu scroll ke bawah.

**AC #4: Total Balance Summary**

* **Given** banyak wallet di carousel,
* **Then** tetap tampilkan "Total Balance" (gabungan semua wallet) secara statis di bagian atas atau sebelum carousel dimulai, agar user tahu total kekayaan mereka dalam sekali lirik.

---

## Implementation Details (Winston's Technical Guide)

### 🛠️ UI Changes

* **Old Code:** `SliverGrid` atau `GridView.builder`.
* **New Code:** `SliverToBoxAdapter` yang membungkus `SizedBox` dengan `ListView.builder` (scrollDirection: Axis.horizontal).
* **Pro Tip:** Gunakan `PageController` atau package `carousel_slider` jika ingin efek kartu yang "magnetis" (Snapping effect) saat di-swipe.

### Tasks / Subtasks

* [ ] **Refactor Widget:** Ubah `WalletGrid` menjadi `WalletCarousel`.
* [ ] **Card Design Update:** Sesuaikan ukuran kartu agar pas dalam mode horizontal (Aspect Ratio sekitar 16:9 atau 3:2).
* [ ] **Sliver Coordination:** Pastikan `CustomScrollView` tetap berjalan mulus saat user men-scroll list transaksi di bawahnya.

### Technical Summary

Tantangan di sini adalah **Padding**. Pastikan kartu pertama memiliki *left padding* yang sejajar dengan judul "Recent Transactions" agar terlihat rapi (Alignment). Gunakan `physics: BouncingScrollPhysics()` untuk memberikan sensasi premium di iOS dan Android.

## Context References

**UX-Spec Update:**

* **Visual Feedback:** Tambahkan *indicator dots* kecil di bawah carousel jika jumlah wallet lebih dari 3, untuk memberi tahu user bahwa ada lebih banyak wallet di samping.
* **Empty State:** Jika wallet hanya 1, tampilkan satu kartu besar yang memenuhi lebar layar.
