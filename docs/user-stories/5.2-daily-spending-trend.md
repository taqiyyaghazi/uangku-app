# Story 5.2: Daily Spending Trend (Line Chart)

**Status:** Ready for Implementation
**Epic:** 5 - Insights & Analytics
**Story Point:** 3 (Data Processing Complexity)

---

## User Story

**As a** user,
**I want** to see a line chart of my daily expenses over the month,
**so that** I can identify specific days or weekends where my spending spikes.

---

## Acceptance Criteria

**AC #1: The Trend Visualization**

* **Given** layar Insights,
* **When** user melakukan scroll di bawah Pie Chart,
* **Then** tampilkan **Line Chart** yang menunjukkan tren pengeluaran harian.

**AC #2: X & Y Axis Logic**

* **Given** grafik garis,
* **Then**:
* **Sumbu X (Horizontal):** Menampilkan tanggal (1 sampai 28/30/31).
* **Sumbu Y (Vertical):** Menampilkan nominal uang.


* **When** ada hari tanpa transaksi,
* **Then** garis harus turun ke titik 0 (bukan memotong kompas ke hari berikutnya).

**AC #3: Interaction (Tooltips)**

* **Given** titik tertentu di grafik,
* **When** user menyentuh (long press/tap) titik tersebut,
* **Then** tampilkan *tooltip* kecil yang menunjukkan detail: "5 Maret: Rp 450.000".

**AC #4: Average Line (Optional/Bonus)**

* **Given** grafik tren,
* **Then** tampilkan garis horizontal putus-putus yang menunjukkan rata-rata pengeluaran harian agar user tahu kapan mereka belanja di atas atau di bawah rata-rata.

---

## Implementation Details

### Tasks / Subtasks

* [ ] **Data Logic (Drift):**
* Buat query di DAO yang menjumlahkan transaksi per hari: `getDailySpendingStream(DateTime month)`.
* **Tips:** Anda perlu melakukan *looping* untuk mengisi hari-hari yang kosong (0) agar grafik tidak patah.


* [ ] **UI Component (fl_chart):**
* Gunakan `LineChart` dari package `fl_chart`.
* Atur `isCurved: true` agar garis terlihat estetik dan modern (Gaya **Ocean Flow**).


* [ ] **Styling:**
* Gunakan gradien di bawah garis (Area Chart) agar tampilan lebih premium.
* Set warna garis ke **Deep Ocean Blue** atau **Teal**.



### Technical Summary

Kesulitan utama di sini adalah **"Gap Filling"**. Database hanya menyimpan tanggal saat ada transaksi. Jika tanggal 10 Maret tidak ada belanja, database tidak akan mengembalikan data. Anda harus membuat *Map* atau *List* berisi seluruh tanggal dalam sebulan di kode Dart, lalu mengisinya dengan data dari database agar grafiknya konsisten dari tanggal 1 sampai akhir bulan.
