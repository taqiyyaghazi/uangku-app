# Story 8.2.1: Cloud Sync for Budgets & Investments

**Status:** Ready for Implementation
**Epic:** 8 - Cloud Synchronization
**Story Point:** 3 (Data Mapping Expansion)

---

## User Story

**As a** user,
**I want** my monthly budget limits and investment snapshots to be synced to the cloud,
**so that** I don't have to re-configure my financial goals and track my assets from scratch on a new device.

---

## Acceptance Criteria

**AC #1: Budget Collection Mapping**

* **Given** user mengatur budget bulanan untuk sebuah kategori,
* **When** data disimpan di SQLite (Drift),
* **Then** buat/perbarui dokumen di Firestore pada path:
`users/{userId}/budgets/{categoryId}`
*Catatan: Menggunakan `categoryId` sebagai ID dokumen memastikan hanya ada satu budget per kategori.*

**AC #2: Investment Snapshot Mapping**

* **Given** user mencatat nilai investasi (Story 5.2),
* **When** snapshot disimpan,
* **Then** kirim data tersebut ke Firestore pada path:
`users/{userId}/investments/{snapshotId}`

**AC #3: Real-time Update Trigger**

* **Given** perubahan pada nilai budget atau data investasi,
* **When** fungsi `update` dipanggil di aplikasi,
* **Then** Firestore harus menerima pembaruan secara asinkron tanpa mengganggu performa UI (Non-blocking).

**AC #4: Snapshot Consistency**

* **Given** data investasi yang memiliki lampiran nominal dan tanggal,
* **Then** pastikan format tanggal di Firestore menggunakan `Timestamp` agar pengurutan grafik investasi di perangkat baru tetap kronologis.

## Implementation Details (Winston's Technical Guide)

### 📂 Data Schema (Firestore)

**Budgets Document:**

```json
{
  "category_id": "cat_123",
  "limit_amount": 1500000.0,
  "period_month": "2026-03",
  "updated_at": "ServerTimestamp"
}

```

**Investment Document:**

```json
{
  "asset_name": "SBN ST011",
  "current_value": 10000000.0,
  "timestamp": "Timestamp",
  "notes": "Dividen cair bulan ini",
  "updated_at": "ServerTimestamp"
}

```

### Tasks / Subtasks

* [ ] **Repository Extension:**
* Tambahkan metode `syncBudgetToCloud(BudgetRow budget)` dan `syncInvestmentToCloud(InvestmentRow inv)`.


* [ ] **Data Converters:**
* Buat extension method `.toFirestoreMap()` pada class model Drift untuk Budget dan Investment.


* [ ] **Sync Hook:**
* Panggil fungsi sync cloud di dalam blok `transaction` database lokal untuk memastikan keduanya berjalan (Atomicity).



---

## Technical Summary

Hati-hati dengan **Budget Bulanan**. Jika aplikasi Anda mendukung budget yang berbeda tiap bulan (misal: Budget Maret vs April), pastikan ID dokumen di Firestore mencakup informasi bulan (misal: `cat123_202603`). Jika budgetnya bersifat tetap/berulang, cukup gunakan `categoryId`.
