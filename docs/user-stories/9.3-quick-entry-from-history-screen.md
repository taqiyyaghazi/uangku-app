# Story 9.3: Quick Entry from History Screen

**Status:** Ready for Implementation
**Epic:** 9 - Power User Experience (UX Refactor)
**Story Point:** 2 (Simple Navigation & Context Passing)

---

## User Story

**As a** user performing manual reconciliation,
**I want** to be able to add a new transaction directly from the History/List screen,
**so that** I don't have to navigate back to the Dashboard and lose my current filter context.

---

## Acceptance Criteria

**AC #1: Floating Action Button (FAB) on History**

- **Given** layar History/List Transaksi,
- **When** user sedang berada di layar tersebut,
- **Then** tampilkan Floating Action Button (FAB) dengan icon (+) yang mencolok di pojok kanan bawah.

**AC #2: Context-Aware Default Wallet**

- **Given** user sedang memfilter daftar transaksi berdasarkan **Wallet A**,
- **When** user menekan tombol (+) dari layar History tersebut,
- **Then** buka form "Add Transaction" dengan **Wallet A** sudah terpilih secara otomatis sebagai _default_.

**AC #3: Seamless Return**

- **Given** user selesai menyimpan transaksi baru yang ditambah dari History,
- **When** form ditutup (Success),
- **Then** kembalikan user ke layar History dan **otomatis perbarui** (refresh) daftar transaksi agar inputan baru langsung terlihat di list.

**AC #4: Consistent Visual Language**

- **Given** tombol (+) di Dashboard dan History,
- **Then** gunakan desain, warna, dan animasi yang identik agar user tidak bingung.

## Implementation Details (Winston's Technical Guide)

### 🛠️ Navigation Logic

Anda perlu mengirim parameter opsional ke layar `AddTransactionScreen`:

```dart
// Di HistoryScreen
floatingActionButton: FloatingActionButton(
  onPressed: () {
    Navigator.push(context, MaterialPageRoute(
      builder: (context) => AddTransactionScreen(
        initialWallet: currentSelectedFilterWallet, // Kirim wallet yang sedang difilter
      ),
    ));
  },
  child: const Icon(Icons.add),
)

```

### Tasks / Subtasks

- [ ] **UI Update:** Tambahkan `FAB` di `Scaffold` milik `HistoryScreen`.
- [ ] **Parameter Update:** Modifikasi constructor `AddTransactionScreen` agar menerima `initialWalletId`.
- [ ] **State Sync:** Pastikan stream database di History Screen memicu _re-build_ saat ada data baru masuk (Biasanya otomatis jika pakai `StreamBuilder` dari Drift).

---

## Technical Summary

Tantangan kecil di sini adalah **UX Flow**. Pastikan saat user menekan "Back" (Batal) dari form input, mereka kembali ke posisi scroll yang sama di History Screen, jangan sampai list-nya ter-reset ke paling atas (Gunakan `AutomaticKeepAliveClientMixin` jika perlu).

## Context References

**UX-Spec Update:**

- **Scroll Behavior:** FAB bisa disembunyikan otomatis saat user men-scroll ke bawah (_Hide on scroll_) agar tidak menutupi detail transaksi paling bawah, dan muncul kembali saat scroll ke atas.
- **Speed:** Menghemat 2-3 kali tap navigasi setiap kali ada data yang terlewat.
