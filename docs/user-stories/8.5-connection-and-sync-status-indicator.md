# Story 8.5: Connection & Sync Status Indicator

**Status:** Ready for Implementation
**Epic:** 8 - Cloud Synchronization
**Story Point:** 2 (UI/UX Logic)

---

## User Story

**As a** user,
**I want** to see my connection status and sync progress,
**so that** I know if my data is safely backed up to the cloud or currently stored only on my device.

---

## Acceptance Criteria

**AC #1: Real-time Connection Monitoring**

* **Given** aplikasi sedang berjalan,
* **When** koneksi internet terputus atau terhubung kembali,
* **Then** perbarui indikator status secara instan tanpa perlu me-refresh halaman.

**AC #2: Non-Intrusive Visual Signal**

* **Given** Dashboard Utama,
* **When** status **Offline**, tampilkan ikon awan dengan garis coret (☁️🚫) atau banner tipis berwarna oranye di bawah App Bar.
* **When** status **Online**, tampilkan ikon awan centang (☁️✅) atau hilangkan banner peringatan.

**AC #3: Syncing State (Optional/Advanced)**

* **Given** internet baru saja terhubung kembali,
* **When** Firestore sedang melakukan *upload* data yang tertunda,
* **Then** tampilkan animasi putar kecil (spinner) pada ikon awan untuk menunjukkan proses sinkronisasi sedang berlangsung.

**AC #4: Consistent State Across Screens**

* **Given** user berpindah dari Dashboard ke History atau Settings,
* **When** status koneksi berubah,
* **Then** indikator harus tetap konsisten dan terlihat di semua layar utama.

---

## Implementation Details (Winston's Technical Guide)

### 📦 Required Library

* `connectivity_plus`: Untuk mendeteksi perubahan status network (WiFi/Mobile Data/None).

### Tasks / Subtasks

* [ ] **Global Listener:**
* Buat `ConnectionProvider` menggunakan `StreamProvider` yang mendengarkan `Connectivity().onConnectivityChanged`.


* [ ] **UI Component:**
* Buat widget `SyncStatusIcon` yang diletakkan di `actions` pada `AppBar`.


* [ ] **Color Palette:**
* **Online:** `Colors.green` atau `Colors.white`.
* **Offline:** `Colors.orange` (Jangan merah, agar tidak terkesan fatal).



### Technical Summary

Gunakan `ConnectivityResult.none` untuk mendeteksi offline. Namun, perlu diingat bahwa terkadang WiFi terhubung tapi "No Internet". Jika ingin lebih akurat, Anda bisa menambahkan *heartbeat* kecil (ping) ke `google.com` atau mengecek `FirebaseFirestore.instance.disableNetwork()` / `enableNetwork()`.

## Context References

**UX-Spec Update:**

* **Tooltips:** Jika user menekan ikon status, tampilkan pesan singkat: *"Your data is synced"* atau *"You are offline. Changes will sync when you're back online."*
* **Animation:** Gunakan `AnimatedSwitcher` saat berganti ikon agar transisinya halus.
