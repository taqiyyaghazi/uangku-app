
# Story 10.2: Multi-Type Transaction Filter (Focus: Investment)

**Status:** Ready for Implementation
**Epic:** 10 - Advanced Discovery & Deep Filtering
**Story Point:** 3 (Query & UI Refinement)

---

## User Story

**As a** user who tracks assets,
**I want** to filter my transaction history by type (specifically Investment),
**so that** I can easily review all my asset-building activities without them being buried under daily expenses.

---

## Acceptance Criteria

**AC #1: Secondary Filter Row (Type Chips)**
- **Given** layar History/Archive,
- **When** layar dibuka,
- **Then** tampilkan baris kedua chip filter di bawah filter "Wallet".
- **Options:** "All", "Income", "Expense", "Transfer", dan **"Investment"**.

**AC #2: Combined Filtering Logic**
- **Given** user sudah memilih salah satu Wallet,
- **When** user menekan chip "Investment",
- **Then** tampilkan hanya transaksi yang bertipe **Investment** yang dilakukan menggunakan **Wallet tersebut**.
- **Note:** Jika "All Wallets" terpilih, tampilkan semua investasi dari semua sumber dana.

**AC #3: Visual Color Coding**
- **Given** daftar transaksi hasil filter,
- **When** tipe "Investment" terpilih,
- **Then** berikan penanda visual yang jelas (misal: nominal berwarna Biru atau Ungu, berbeda dengan Hijau untuk Income dan Merah untuk Expense) agar mudah dibedakan secara cepat.

**AC #4: Persistent Search Integration**
- **Given** filter "Investment" aktif,
- **When** user mencari kata kunci di search bar (misal: "SBN"),
- **Then** hasil yang muncul harus merupakan irisan dari: Wallet (jika ada) + Tipe Investment + Kata Kunci "SBN".

## Implementation Details (Winston's Technical Guide)

### 🛠️ Database Logic (Drift/SQLite)
Perbarui query di DAO Anda untuk menerima parameter `type` opsional:

```dart
Stream<List<Transaction>> watchFilteredTransactions({
  int? walletId,
  TransactionType? type, // Enum: income, expense, transfer, investment
  String? searchQuery,
}) {
  return (select(transactions)
    ..where((t) {
      final predicate = <Expression<bool>>[];
      if (walletId != null) predicate.add(t.walletId.equals(walletId) | t.toWalletId.equals(walletId));
      if (type != null) predicate.add(t.type.equals(type.index));
      if (searchQuery != null) predicate.add(t.notes.contains(searchQuery));
      return buildAnd(predicate);
    }))
    .watch();
}
```

### 📋 UI Component
- Gunakan `SliverPersistentHeader` atau `SliverToBoxAdapter` untuk menaruh baris chip kedua agar tetap rapi saat di-scroll.
- Simpan state filter tipe dalam variabel `selectedType` (nullable).

## Technical Summary
Tantangan utamanya adalah **Logic Transfer**. Pastikan jika user memilih "Transfer", query Anda tetap mengecek kedua kolom (`wallet_id` dan `to_wallet_id`). Namun untuk "Investment", biasanya hanya melibatkan satu wallet asal, jadi pastikan pemetaan tipe di database sudah konsisten sejak Story 5.2.
