# Story 8.4: Flutter Flavors Setup (Environment Separation)

**Status:** Ready for Implementation
**Epic:** 8 - Cloud Synchronization
**Story Point:** 5 (Build Configuration Complexity)

---

## User Story

**As a** developer,
**I want** to have separate environments for Development and Production,
**so that** I can test new features (like Cloud Sync) using a test database without affecting real user data.

---

## Acceptance Criteria

**AC #1: Android Identity Separation**

- **Given** konfigurasi Gradle,
- **When** melakukan build flavor `dev`, maka `applicationId` menjadi `com.taqiyyaghazi.uangku.dev` dan nama aplikasi "Uangku Dev".
- **When** melakukan build flavor `prod`, maka `applicationId` tetap `com.taqiyyaghazi.uangku` dan nama aplikasi "Uangku".

**AC #2: Firebase Configuration Separation**

- **Given** dua lingkungan berbeda,
- **Then** gunakan file `google-services.json` yang berbeda untuk masing-masing flavor:
- Dev: Terhubung ke Firebase Project "Uangku-Dev".
- Prod: Terhubung ke Firebase Project "Uangku-Prod".

**AC #3: Entry Point Separation (Main.dart)**

- **Given** project Flutter,
- **Then** sediakan dua entry point berbeda: `lib/main_dev.dart` dan `lib/main_prod.dart`.
- **When** dijalankan, masing-masing harus menginisialisasi Firebase dengan opsi (`FirebaseOptions`) yang sesuai dengan lingkungannya.

**AC #4: Visual Distinction**

- **Given** aplikasi terinstal di HP,
- **When** dalam mode `dev`, gunakan icon aplikasi yang memiliki label "DEV" (atau warna berbeda) agar mudah dibedakan secara visual oleh developer/tester.

## Implementation Details (Winston's Technical Guide)

### 1. Folder Structure untuk Firebase

Anda harus memindahkan `google-services.json` ke folder sumber spesifik flavor di Android:

```text
android/app/src/
  ├── dev/
  │   └── google-services.json (Project Dev)
  ├── prod/
  │   └── google-services.json (Project Prod)
  └── main/
      └── (google-services.json dihapus dari sini)

```

### 2. Gradle Setup (`android/app/build.gradle.kts`)

```kotlin
android {
    ...
    flavorDimensions += "app"

    productFlavors {
        create("dev") {
            dimension = "app"
            applicationIdSuffix = ".dev"
            resValue("string", "app_name", "Uangku Dev")
        }
        create("prod") {
            dimension = "app"
            resValue("string", "app_name", "Uangku")
        }
    }
}

```

### 3. Entry Point Logic

Buat class `AppConfig` untuk menyimpan status lingkungan saat ini:

```dart
enum Environment { dev, prod }

class AppConfig {
  static Environment environment = Environment.prod;
  static bool get isDev => environment == Environment.dev;
}

```

## Technical Summary

Poin krusial di sini adalah **Firebase Options**. Jika Anda menggunakan `flutterfire configure`, pastikan Anda menjalankan perintah tersebut dua kali untuk dua project berbeda dan simpan outputnya dalam file terpisah (misal `firebase_options_dev.dart` dan `firebase_options_prod.dart`).
