# Story 11.1: Secure Logout & Local Data Cleansing

**Status:** Completed
**Epic:** 11 - Data Privacy & Session Security
**Story Point:** 3 (Critical Logic)

---

## User Story

**As a** user who shares a device or has multiple accounts,
**I want** the app to completely wipe my local financial data upon logout,
**so that** if a different user logs in, my data doesn't leak into their account and they can start with a fresh state.

---

## Acceptance Criteria

**AC #1: Atomic Database Wipe**

- **Given** user menekan tombol "Logout",
- **When** proses logout dimulai,
- **Then** aplikasi wajib menjalankan perintah `DELETE` pada semua tabel lokal (Transactions, Wallets, Categories, Budgets, Investments) secara bersamaan (Atomic).

**AC #2: Shared Preferences Reset**

- **Given** aplikasi menyimpan metadata (misal: `last_sync_time`, `selected_theme`, atau `user_preferences`),
- **When** logout berhasil,
- **Then** hapus semua data di `SharedPreferences` agar akun baru tidak menggunakan pengaturan akun lama.

**AC #3: Session Termination**

- **Given** proses pembersihan data lokal selesai,
- **Then** panggil fungsi `signOut()` dari Firebase Auth dan Google Sign-In, lalu arahkan user kembali ke layar Login.

**AC #4: Block UI during Cleansing**

- **Given** proses penghapusan data,
- **When** sedang berlangsung,
- **Then** tampilkan _Loading Overlay_ yang tidak bisa di-_dismiss_ oleh user untuk mencegah kegagalan proses karena aplikasi ditutup paksa.

---

## Implementation Details (Winston's Technical Guide)

### 🛠️ Drift Database Reset

Tambahkan method khusus di class database Anda untuk menghapus semua data tanpa menghapus skema:

```dart
Future<void> deleteAllLocalData() {
  return transaction(() async {
    // Hapus dengan urutan yang benar (menghindari Foreign Key constraint)
    await delete(transactions).go();
    await delete(budgets).go();
    await delete(investments).go();
    await delete(wallets).go();
    await delete(categories).go();
  });
}
```

### 📋 Service Integration

Gabungkan logika ini di dalam `AuthService`:

```dart
Future<void> performSecureLogout() async {
  try {
    // 1. Wipe SQLite
    await appDatabase.deleteAllLocalData();
    // 2. Wipe Prefs
    final prefs = await SharedPreferences.getInstance();
    await prefs.clear();
    // 3. Firebase Sign Out
    await FirebaseAuth.instance.signOut();
    await GoogleSignIn().signOut();
  } catch (e) {
    // Log error to Crashlytics
    FirebaseCrashlytics.instance.log("Logout failed: $e");
  }
}
```

---

## Technical Summary

Masalah utamanya bukan di Firebase, tapi di **Local Persistence**. Drift (SQLite) bersifat permanen di penyimpanan HP. Jika tidak dihapus manual, dia akan tetap di sana selamanya. Dengan menjalankan `deleteAllLocalData()` sebelum `signOut`, kita memastikan akun baru akan memicu Story 8.3 (Initial Data Pull) dari Firestore-nya sendiri secara bersih.
