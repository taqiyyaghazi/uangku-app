# Story 10.2: Core Transaction Type Filter

**Status:** Ready for Implementation
**Epic:** 10 - Advanced Discovery & Deep Filtering
**Story Point:** 2 (UI Filter Expansion)

---

## User Story

**As a** user,
**I want** to filter my transaction history by its core type (Expense, Income, or Transfer),
**so that** I can quickly separate my actual spending from internal money movements (Transfers) or earnings.

---

## Acceptance Criteria

**AC #1: Secondary Filter Row (The Core Trio)**
- **Given** layar History/Archive,
- **When** user ingin mempersempit pencarian,
- **Then** tampilkan baris chip filter kedua (di bawah filter Wallet) dengan pilihan:
  - **All** (Default)
  - **Expense** (Uang Keluar)
  - **Income** (Uang Masuk)
  - **Transfer** (Pindah Buku antar Wallet)

**AC #2: Multi-Dimensional Filtering**
- **Given** user sudah memilih salah satu Wallet,
- **When** user mengaktifkan filter "Transfer",
- **Then** tampilkan hanya transaksi pindah buku yang melibatkan wallet tersebut.
- **Goal:** Memudahkan user mengecek apakah pemindahan saldo antar bank sudah tercatat semua.

**AC #3: Visual Consistency**
- **Given** chip filter yang dipilih,
- **Then** gunakan warna identitas masing-masing tipe:
  - **Expense:** Merah 🔴
  - **Income:** Hijau 🟢
  - **Transfer:** Biru atau Abu-abu 🔵

**AC #4: Persistent Search Integration**
- **Given** filter tipe aktif (misal: "Expense"),
- **When** user mengetik di search bar (misal: "Saham" atau "Reksadana"),
- **Then** hasil yang muncul hanya transaksi pengeluaran (Expense) yang memiliki catatan kata tersebut.

---

## Implementation Details (Winston's Technical Guide)

### 🛠️ UI Refinement
Ganti baris filter tunggal Anda dengan `Column` yang berisi dua `SingleChildScrollView` (Horizontal):
1. **Row 1:** Wallet Chips.
2. **Row 2:** Type Chips (Expense, Income, Transfer).

### 📋 Database Logic (Drift/SQLite)
Query Anda sekarang menjadi lebih *powerful* karena menggabungkan dua filter:

```dart
// Filter di level UI
final filteredList = allTransactions.where((t) {
  final matchWallet = selectedWalletId == null || t.walletId == selectedWalletId || t.toWalletId == selectedWalletId;
  final matchType = selectedType == null || t.type == selectedType;
  return matchWallet && matchType;
}).toList();
```



---

## Technical Summary
Dengan memisahkan **Transfer** dari **Expense**, user yang merasa "kesulitan mengecek transaksi investment" (seperti di feedback) bisa langsung memilih filter **Transfer** (jika mereka mencatat investasi sebagai pindah buku ke wallet investasi) atau **Expense** lalu mencari kata kunci asetnya. Ini jauh lebih fleksibel!

## Context References

**UX-Spec Update:**
