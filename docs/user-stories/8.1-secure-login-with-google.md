# Story 8.1: Secure Login with Google (Firebase Auth)

**Status:** Ready for Implementation
**Epic:** 8 - Cloud Synchronization
**Story Point:** 3 (Identity Management)

---

## User Story

**As a** user,
**I want** to sign in with my Google account,
**so that** my financial data can be safely linked to my identity and backed up automatically.

---

## Acceptance Criteria

**AC #1: Authentication UI**

- **Given** aplikasi pertama kali dibuka (atau menu Settings),
- **When** user belum login,
- **Then** tampilkan tombol "Sign in with Google" yang mengikuti _Google Branding Guidelines_.

**AC #2: Success Authentication**

- **Given** user menekan tombol login dan memilih akun Google,
- **When** proses autentikasi berhasil,
- **Then** simpan profil dasar user (Nama, Foto, Email) dan arahkan ke Dashboard.

**AC #3: User Profile in Settings**

- **Given** user sudah login,
- **When** membuka menu Settings,
- **Then** tampilkan info profil (Foto & Email) dan tombol "Sign Out".

**AC #4: Persistent Session**

- **Given** user sudah login sebelumnya,
- **When** aplikasi dibuka kembali,
- **Then** user tidak perlu login ulang (Auto-login).

---

## Implementation Details (Winston's Technical Guide)

### 📦 Required Libraries

- `firebase_auth`: Untuk manajemen session user.
- `google_sign_in`: Untuk memicu pop-up pilihan akun Google di Android/iOS.

### Tasks / Subtasks

- [ ] **Firebase Console Setup:**
- Aktifkan **Authentication** > **Sign-in Method** > **Google**.
- Masukkan **SHA-1 Fingerprint** dari laptop Anda dan dari GitHub Actions (Secrets) ke Firebase Project Settings.

- [ ] **UI Component:**
- Buat `AuthWrapper` yang mengecek `FirebaseAuth.instance.authStateChanges()`.
- Jika user ada -> Tampilkan Dashboard. Jika tidak ada -> Tampilkan Login Screen.

- [ ] **Data Linking:**
- Simpan `uid` (User ID) dari Firebase Auth untuk digunakan sebagai kunci utama di Firestore nanti (Story 8.2).

### Technical Summary

Tantangan terbesar di sini adalah **SHA-1 Fingerprint**. Tanpa ini, Google Sign-In akan error (Developer Error 10). Pastikan SHA-1 dari `debug.keystore` (lokal) dan `upload-keystore.jks` (untuk rilis) keduanya terdaftar di Firebase.
