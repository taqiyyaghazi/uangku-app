# Story 5.1: The Spending Pie (Category Distribution)

**Status:** Ready for Implementation
**Epic:** 5 - Insights & Analytics
**Story Point:** 3 (Visual Complexity)

---

## User Story

**As a** user,
**I want** to see a visual breakdown of my spending by category,
**so that** I can identify which areas of my life are consuming the most money.

---

## Acceptance Criteria

**AC #1: The Insight Tab/Section**

* **Given** Dashboard utama,
* **When** user melakukan navigasi ke tab baru bernama "Insights" atau menekan "View Analytics",
* **Then** tampilkan layar khusus statistik.

**AC #2: Interactive Pie Chart**

* **Given** data transaksi di bulan berjalan,
* **When** layar dibuka,
* **Then** tampilkan **Pie Chart** (atau Donut Chart) yang membagi total pengeluaran berdasarkan kategori (Food, Transport, Bills, etc).

**AC #3: Dynamic Legend & Percentage**

* **Given** grafik pie,
* **When** sebuah potongan (slice) ditekan atau dilihat,
* **Then** tampilkan nama kategori, jumlah nominal, dan persentasenya (misal: "Food: Rp 1.200.000 - 35%").

**AC #4: Time Range Filter**

* **Given** layar Insight,
* **When** user ingin melihat bulan lalu,
* **Then** sediakan *dropdown* atau *date picker* untuk memilih periode laporan (Bulan ini, Bulan lalu, atau Custom).

---

## Implementation Details (Tips untuk Dev Agent)

### Tasks / Subtasks

* [ ] **Library Selection:**
* Gunakan package `fl_chart` (sangat fleksibel) atau `syncfusion_flutter_charts`.


* [ ] **Data Aggregation (Drift):**
* Buat query di DAO: `getCategorySpending(DateTime month)`.
* Gunakan `SUM(amount)` dan `GROUP BY category_id`.


* [ ] **UI Component:**
* Buat `SpendingPieChart` widget.
* Pastikan warna setiap potongan *slice* sesuai dengan warna kategori yang sudah Anda set di **Story 4.6**.


* [ ] **Empty State:**
* Jika tidak ada transaksi di bulan tersebut, tampilkan pesan: *"Belum ada data untuk periode ini."*



### Technical Summary

Tantangan di sini adalah **Data Mapping**. Anda harus mengambil data dari tabel `Transactions`, menjumlahkannya berdasarkan `category_id`, lalu menggabungkannya dengan tabel `Categories` untuk mendapatkan nama dan warnanya. Gunakan **StreamBuilder** agar grafik otomatis update jika ada transaksi baru yang masuk.
