# Story 13.1: AI Accuracy Tracking via Firebase Analytics

**Status:** Ready for Implementation
**Epic:** 13 - Financial Health & AI Optimization
**Story Point:** 2 (Analytics & Event Tracking)

---

## User Story

**As a** Product Manager,
**I want** to log every AI-generated transaction along with its accuracy status,
**so that** I can analyze the AI's performance and improve the prompt logic based on real-world mismatch data.

---

## Acceptance Criteria

**AC #1: Event Trigger on Save**

- **Given** user sedang menggunakan fitur Scan Receipt atau NLP Chat,
- **When** user menekan tombol "Simpan" (Save),
- **Then** kirim event `ai_performance_v1` ke Firebase Analytics.

**AC #2: Accuracy Comparison Logic**

- **Given** transaksi yang disarankan oleh AI,
- **When** data akan dikirim ke Analytics,
- **Then** bandingkan `suggested_category` (dari Gemini) dengan `final_category` (pilihan akhir user).
- **Metric:** Kirim parameter `is_correct` (1 jika sama, 0 jika berbeda).

**AC #3: Essential Metadata Parameters**

- **Given** setiap event yang dikirim,
- **Then** wajib menyertakan parameter berikut:
  - `method`: (scan_receipt / nlp_chat)
  - `ai_suggested_category`: Kategori awal dari AI.
  - `user_final_category`: Kategori yang disimpan user.
  - `is_correct`: Flag keberhasilan (Integer 1/0).

**AC #4: Non-Intrusive Operation**

- **Given** proses logging,
- **When** terjadi error pada pengiriman log (misal: internet tidak stabil),
- **Then** jangan biarkan error tersebut menghentikan proses penyimpanan transaksi ke database lokal (Silent error handling).

## Implementation Details (Winston's Technical Guide)

### 🛠️ Analytics Structure

Gunakan wrapper class untuk konsistensi data. Pastikan nama parameter tidak lebih dari 40 karakter sesuai limit Firebase.

```dart
// Code Implementation Snippet
Future<void> logAiSuccess({
  required String method,
  required String aiCategory,
  required String finalCategory,
}) async {
  await FirebaseAnalytics.instance.logEvent(
    name: 'ai_performance_v1',
    parameters: {
      'method': method,
      'ai_cat': aiCategory,
      'final_cat': finalCategory,
      'is_correct': aiCategory == finalCategory ? 1 : 0,
    },
  );
}
```

### 📋 Post-Implementation Task (Firebase Console)

Setelah kodingan ini rilis, Anda **WAJIB** melakukan hal berikut agar data bisa diolah:

1.  Buka **Firebase Console** > **Analytics** > **Custom Definitions**.
2.  Daftarkan `method`, `ai_cat`, `final_cat`, dan `is_correct` sebagai **Custom Dimensions**.

## Technical Summary

Kita menggunakan **Firebase Analytics** sebagai gudang data karena sifatnya yang _asynchronous_ dan tidak membebani performa aplikasi. Data ini nantinya bisa kita tarik ke **Google Looker Studio** untuk membuat diagram pie "Akurasi Bob" yang bisa dipantau secara real-time.
