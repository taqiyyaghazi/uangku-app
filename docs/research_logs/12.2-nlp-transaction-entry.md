# Research Log: NLP Transaction Entry (Chat-to-Record)

## Overview
This feature allows users to record transactions via natural language, either by typing or speaking. It relies on the Gemini model to parse the input and map it to `amount`, `type`, `wallet`, `category`, and `note`.

## Current Implementation Context
- The app currently uses a FAB on the `DashboardScreen` that opens a `QuickEntrySheet`.
- A `GeminiScannerService` already exists (`lib/features/transaction/services/gemini_scanner_service.dart`) which uses the `google_generative_ai` package and the `gemini-2.5-flash-lite` model for receipt scanning.
- We need to integrate a chat-like text field. The story suggests either at the bottom of the dashboard or inside an expandable FAB. We can replace the existing FAB with a custom bottom input bar or an expandable FAB, or add the text field inside `QuickEntrySheet`. However, the AC states: "di bagian bawah dashboard atau di dalam FAB yang bisa di-expand", so a persistent chat bar or expandable FAB at the bottom of the `DashboardScreen` seems appropriate.
- Speech-to-text requires the `speech_to_text` package, which is currently missing from `pubspec.yaml` and needs to be added.

## Dependencies & Technologies
- **google_generative_ai**: Already installed and used.
- **speech_to_text**: Needs to be installed. Provides voice recognition capabilities. We'll need to configure iOS `Info.plist` and Android `AndroidManifest.xml` with appropriate microphone and speech recognition permissions.

## UI/UX Flow
1. User taps on the chat input bar or microphone icon on the Dashboard.
2. User speaks or types a sentence (e.g., "Tadi bayar kopi 25rb pakai Gopay").
3. A `TypeWriter` animation plays (Bob is thinking).
4. A Gemini API call is made with a context-heavy prompt containing the user's available wallets and categories.
5. The API returns a JSON structure with the extracted fields.
6. A "Smart Confirmation Dialog" (Summary Card) appears: "Bob mencatat: Pengeluaran 25.000 untuk Kopi dari Gopay. Benar?".
7. The user can tap "Simpan" to save directly or "Edit" to open the manual form (e.g., `QuickEntrySheet` pre-filled).

## Prompt Engineering
The prompt needs to explicitly list:
- User's available wallets.
- User's available categories.
- Default wallet (for defaulting if not mentioned).
- Instructions to return ONLY JSON.
- Example JSON output structure.

## Integration Points
- `DashboardScreen`: Modify the bottom area or FAB to host the NLP input field.
- `GeminiScannerService`: Add a new method `analyzeTransactionText(String text, List<Category> categories, List<Wallet> wallets, Wallet defaultWallet)` or create a new `GeminiNlpService`.
- `TransactionRepository`: Use it to save the transaction once confirmed.
- `speech_to_text` integration for the microphone button.

## Open Questions / Decisions
- Should the chat bar be persistent at the bottom of the dashboard, replacing the standard FAB, or should it sit alongside it? A persistent bottom input bar (like a messaging app) would provide the most seamless UX.
- We need to handle incomplete data. If Gemini cannot determine a required field, the confirmation dialog should probably force the user into the "Edit" flow (the `QuickEntrySheet`), pre-filled with whatever was extracted.
