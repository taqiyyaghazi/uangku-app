# Research Log: Quick Entry from History Screen

## 📋 Analysis

- **`TransactionsArchiveScreen`**:
  - Uses a `CustomScrollView` with slivers.
  - Currently has no Floating Action Button (FAB).
  - Tracks `selectedWalletId` from `selectedWalletFilterProvider`.
  - Filter logic is reactive via Riverpod.

- **`QuickEntrySheet`**:
  - A `ConsumerStatefulWidget` showing a bottom sheet.
  - Currently defaults wallet selection to the first available wallet in the list.
  - Needs a way to accept an `initialWalletId`.

## 🎯 Proposed Solution

1. **Modify `QuickEntrySheet`**:
   - Add `int? initialWalletId` to constructor and `static show()`.
   - In `initState`, if `initialWalletId` is provided, use it to set `_selectedWalletId`.

2. **Modify `TransactionsArchiveScreen`**:
   - Add a `FloatingActionButton` to the `Scaffold`.
   - Pass the current `selectedWalletId` from the filter to `QuickEntrySheet.show()`.
   - Visual consistency: Use the same teal theme color as the Dashboard FAB.

3. **UX Improvement**:
   - Implement "Hide on Scroll" for the FAB to avoid obscuring transaction details.
