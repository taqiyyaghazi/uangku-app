# Research Log: Searchable Picker for Categories & Wallets

## 📋 Analysis of Current Implementation

- **`QuickEntrySheet`:**
  - Uses `ListView.separated` (horizontal) with `ChoiceChip` for both wallets and categories.
  - Category list is dynamic based on selected transaction type (Income/Expense).
  - Logic for auto-selecting the first item if none is selected.
  - Wallets are fetched from `walletsProvider`.
  - Categories are fetched from `categoriesByTypeProvider`.

- **`TransactionDetailSheet`:**
  - Uses a similar `ListView` + `ChoiceChip` pattern in edit mode.
  - Only has Category selector (Wallet is fixed in edit mode based on current UI).
  - Uses `StreamBuilder` for categories.

## 🎯 Acceptance Criteria Analysis

- **AC #1: Bottom Sheet Trigger**
  - Trigger: Tap on "Category" or "Wallet" area.
  - UI: `ModalBottomSheet` (50-70% height).
  - Current UI uses horizontal chips; will change to a read-only field or a specialized button that looks like a field.

- **AC #2: Real-time Search Bar**
  - Auto-focus on open.
  - Instant filtering based on name.

- **AC #3: Visual List Organization**
  - Vertical list (`ListTile`).
  - Show Icon, Name, and Balance (for wallets).
  - Highlight matching text.

- **AC #4: Rapid Selection & Auto-Close**
  - Tap item -> select -> pop.

- **UX Spec: Empty State**
  - "Add New Category: [Text]" button if no match.

- **UX Spec: Recent Items**
  - Show top 3 frequently used categories first. This requires tracking usage frequency or just showing the last used ones. I'll check if there's an existing service for this.

## 🛠️ Proposed Solution

1. **`SearchablePickerSheet` Component:**
   - A generic widget that can handle any item with a name, icon, and optional extra info (like balance).
   - Internal `TextEditingController` for search.
   - Use `ListView.builder` for the list.
   - Use a custom `SearchField` widget at the top.

2. **Integration:**
   - Replace the horizontal chip lists in `QuickEntrySheet` and `TransactionDetailSheet` with a "Picker Trigger" (e.g., a `ListTile` or `InkWell` styled as a field).
   - Implement the "Add New Category" logic by invoking the category creation flow.

3. **Recent Items Logic:**
   - For now, I can implement a simple "Recent Categories" by fetching the latest transactions and extracting unique category IDs.

## 🔬 Technology Search Results

- **Highlight Text Matching:**
  - Can use `Text.rich` with `TextSpan` to highlight substrings.

- **Search Filtering:**
  - Simple `.where((item) => item.name.toLowerCase().contains(query.toLowerCase()))`.
