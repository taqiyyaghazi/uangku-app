# Research Log: Core Transaction Type Filter (Story 10.2)

## Current Implementation State

### Navigation & State
- File: `lib/core/di/providers.dart`
- Wallet filtering is currently managed by `selectedWalletFilterProvider` (a `StateNotifierProvider` or `NotifierProvider` holding `int?`).
- `allTransactionsProvider` watches both `transactionRepositoryProvider` and `selectedWalletFilterProvider` to fetch filtered data.

### Database Layer
- File: `lib/data/repositories/transaction_repository.dart` & `lib/data/daos/drift_transaction_repository.dart`
- `watchAllTransactions({int? walletId})` currently builds a query using Drift. It already handles optional `walletId` filtering at the database level.
- `TransactionType` enum is stored in Drift as a `TextColumn` via `textEnum<TransactionType>()`. The types are `income`, `expense`, and `transfer` (as defined in `lib/data/tables/transactions_table.dart`).

### UI Layer
- File: `lib/features/transaction/screens/transactions_archive_screen.dart`
- Displays full transaction history grouped by month.
- The top header (`SliverAppBar.bottom`) contains:
  1. A `CupertinoSearchTextField` for text-based filtering against note/category (`_searchQuery`).
  2. An `ActionChip` that when tapped opens a `SearchablePickerSheet` to pick a wallet filter.
- Text search filtering is currently done strictly at the UI-level using `TransactionGroupingLogic.filterBySearchQuery`.

## Technical Plan & Decisions

We have two options to filter by `TransactionType`:
1. **Database Level:** Modify `watchAllTransactions` to accept `TransactionType? type` and add a `where` clause (`_db.transactions.type.equals(type.name)`).
2. **UI Level:** Do it purely in Flutter just like `_searchQuery`. (Winston's guide gave the UI-level implementation as an example, but database level is more robust and consistent since we already do wallet filtering at DB level).

**Decision:** We will combine UI/DB level. Since `allTransactionsProvider` already filters by wallet at DB level, we will extend it to also filter by type at DB level for performance, which will naturally stream the correct bounded dataset to the UI. The text search will continue to filter on the UI side.

### Key Modifications Needed:

1. **`lib/core/di/providers.dart`**:
   - Create `SelectedTypeFilterNotifier` & `selectedTypeFilterProvider` storing `TransactionType?`.
   - Update `allTransactionsProvider` to ref.watch this new provider.

   - Update `watchAllTransactions({int? walletId, TransactionType? type})`.
   - Add Drift where clause: `if (type != null) query.where(_db.transactions.type.equals(type.name));`.

3. **`lib/features/transaction/screens/transactions_archive_screen.dart`**:
   - Update header to include a row of type chips (`All`, `Expense` (Red), `Income` (Green), `Transfer` (Blue)).
   - Implement chip selection logic to update `selectedTypeFilterProvider`.
   - Keep text search working perfectly over the already-filtered stream!
