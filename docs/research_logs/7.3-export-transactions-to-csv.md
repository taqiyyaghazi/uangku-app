# Research Log: Export Transactions to CSV (Feature 7.3)

## Topics Searched

- Export list to CSV Dart (`csv` package)
- Get temp directory Flutter (`path_provider` package)
- Share files iOS Android Flutter (`share_plus` package)

_Note: Relying on training data for API signatures as well since these are well-established libraries._

## Findings

### CSV Generation

The `csv` package provides a robust way to convert lists to CSV strings.

```dart
import 'package:csv/csv.dart';

List<List<dynamic>> rows = [
  ['Date', 'Amount', 'Category', 'Wallet', 'Type', 'Notes'],
  ['2023-10-01', 50000, 'Food', 'Cash', 'Expense', 'Lunch'],
];
String csv = const ListToCsvConverter().convert(rows);
```

### File System Access

We need to generate a temporary CSV file before sharing. `path_provider` gets us the proper sandboxed temporary directory to write files without requiring complex OS storage permissions on newer versions of Android/iOS.

```dart
import 'package:path_provider/path_provider.dart';
import 'dart:io';

final directory = await getTemporaryDirectory();
```

### Native Share Sheet

We use `share_plus` to trigger the native share dialog.

```dart
import 'package:share_plus/share_plus.dart';

final result = await Share.shareXFiles(
  [XFile(path)],
  text: 'Here is your transaction export',
);

if (result.status == ShareResultStatus.success) {
  // Share was successful
}
```

## Architecture Decisions

No major new architecture decisions beyond adding a new pure service (`ExportService`) for the conversion logic and mixing the I/O parts within the presentation application layer or an infrastructure facade. We'll use the existing abstraction patterns.

- The formatting of the CSV is a pure function (`generateCsv`).
- The reading and writing of files is I/O, which must be mockable, but it is standard to inject these or just do it in a lightweight Service that interacts with `share_plus`, which can be wrapped in a repository interface if needed, or tested via integration tests. For simplicity, we'll extract the CSV string building into a pure function unit tested in isolation, and the file saving/sharing in the provider logic, allowing mockable dependencies.
