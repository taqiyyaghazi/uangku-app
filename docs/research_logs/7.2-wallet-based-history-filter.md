# Research Log: Wallet-Based History Filter (Story 7.2)

## Context

Goal is to implement AC #1-4 from story 7.2.
Requires adding a `walletId` filter that handles transfers (`to_wallet_id`).

## Existing Implementation

- **Database:** Drift SQLite. `TransactionsTable` has `walletId` and `toWalletId`.
- **Repository:** `DriftTransactionRepository` handles queries. `watchAllTransactions()` currently returns a stream of all transactions.
- **State:** Riverpod `providers.dart` exposes `allTransactionsProvider` (`StreamProvider`).
- **UI:** `TransactionsArchiveScreen` consumes `allTransactionsProvider`. Search is handled locally in the UI (`TransactionGroupingLogic.filterBySearchQuery`).

## Plan

Instead of handling the wallet filter locally in logic, doing it at the database query level (`watchAllTransactions`) is cleaner and more performant for large datasets.

1.  **Repository Interface:** Add optional named parameter `{int? walletId}` to `watchAllTransactions`.
2.  **Drift Implementation:**

    ```dart
    final query = _db.select(_db.transactions).join([...])
      ..orderBy([OrderingTerm.desc(_db.transactions.date)]);

    if (walletId != null) {
      query.where(
        _db.transactions.walletId.equals(walletId) |
        _db.transactions.toWalletId.equals(walletId)
      );
    }
    ```

3.  **State Management:** Add `selectedWalletFilterProvider = StateProvider<int?>((ref) => null)`. Update `allTransactionsProvider` to watch this state and pass it to the repository.
4.  **UI:** Add `ChoiceChip` list. Handle empty states carefully (general empty vs wallet empty vs search empty).
