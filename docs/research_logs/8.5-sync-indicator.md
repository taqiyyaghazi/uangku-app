# Research Log: Connection & Sync Status Indicator

## Objective

Detect real-time network status changes and provide visual feedback to the user regarding cloud synchronization health.

## Findings

### Connectivity Monitoring

- `connectivity_plus` is the standard library for this.
- It provides a stream: `Connectivity().onConnectivityChanged`.
- Note: `connectivity_plus` v6.0+ returns a `List<ConnectivityResult>`.
- `ConnectivityResult.none` indicates no connection.

### UI Integration Points

1. **Insights & Portfolio Screens**: Use standard `AppBar`. The indicator can be added to `actions`.
2. **Dashboard Screen**: Uses `DashboardHeader`. The indicator should be placed next to the `UserAvatarButton`.
3. **Global Visibility**: Since `MainShell` manages the bottom nav but not the header, each screen must individually host the indicator (or a shared layout component could be introduced, but sticking to existing patterns is safer for now).

### Sync State Mapping

- `SyncStatus.syncing`: Show rotating spinner.
- `Offline`: Show `Icons.cloud_off` (Orange).
- `Online & Idle`: Show `Icons.cloud_done` or `Icons.check_circle_outline` (Green/White).

## Proposed Architecture

1. **ConnectionProvider**: A `StreamProvider<List<ConnectivityResult>>` that listens to `Connectivity().onConnectivityChanged`.
2. **SyncStatusIcon**: A reusable `ConsumerWidget` that:
   - Watches `ConnectionProvider`.
   - Watches `syncStatusProvider`.
   - Displays appropriate icon/tooltip.
   - Uses `AnimatedSwitcher` for smooth transitions.

## Potential Challenges

- **False Positives**: `ConnectivityResult` tells if you are connected to a network, but not if that network has internet. Firebase already handles offline persistence well, so we can also check `FirebaseFirestore.instance.disableNetwork()`/`enableNetwork()` if we want deeper control, but for AC #1, `connectivity_plus` is sufficient.
