# Research Log: Dashboard Wallet Layout Refactor (Carousel View)

## 📋 Analysis of Current Implementation

- **`DashboardScreen`:**
  - Uses a `CustomScrollView` with various slivers.
  - The "Total Balance" summary is already present at the top through `DashboardHeader`.
  - The wallet grid is currently implemented using `WalletGrid` widget, which is a `SliverPadding` wrapping a `SliverGrid`.
  - It shows `2` columns with a fixed child aspect ratio (`1.35`).

- **`WalletGrid`:**
  - A simple wrapper around `SliverGrid`.
  - Always shows all wallets in a 2-column grid.
  - Includes an `_AddWalletCard` at the end of the list.

- **`WalletCard`:**
  - A simple card showing the wallet's icon, type, name, and balance.
  - Uses a fixed padding and internal alignment.

## 🎯 Acceptance Criteria Analysis

- **AC #1: Compact Horizontal Carousel**
  - Trigger: > 2 wallets.
  - New Widget: `WalletCarousel`.
  - Height: Fixed (approx. `120px - 150px`).
  - Layout: `ListView` with `scrollDirection: Axis.horizontal`.

- **AC #2: Balanced Information Display**
  - Minimalist info: Name, Icon, Saldo.
  - UX: Proper width to avoid truncated balance.

- **AC #3: Better "Recent Transactions" Visibility**
  - Reducing the wallet section's vertical space will help show more transactions above the fold.

- **AC #4: Total Balance Summary**
  - Already present, but I should ensure its position satisfies the "static at top" requirement.

- **UX Spec: Indicator Dots**
  - Add for > 3 wallets.

## 🛠️ Proposed Solution

1. **`WalletCarousel` Component:**
   - Use `ListView.builder` for horizontal scrolling.
   - Use `PageController` or a simple `ListView` with snapping if needed. Since the task mentions `carousel_slider`, I'll check if it's available in `pubspec.yaml`.
   - Wait, `pubspec.yaml` doesn't have `carousel_slider`. I'll implement it manually with a `ListView` and maybe `PageScrollPhysics` or similar if snapping is required.
   - The "Add Wallet" card will be the last item in the list.

2. **Integration:**
   - Modify `DashboardScreen` to conditionally use `WalletCarousel` or `WalletGrid` (or maybe just use carousel always?).
   - AC #1 says "When having more than 2 wallets, replace Grid with Carousel". If <= 2, should I keep the Grid? Or just one big card? AC #1 says "If wallet only 1, show one big card filling the screen width".

3. **Styling:**
   - Ensure `ListView` padding matches the "Recent Transactions" title padding.
   - The height of the `ListView` should be about `140px`.

## 🔬 Technology Search Results

- **Carousel Implementation in Flutter:**
  - A `ListView` with `physics: PageScrollPhysics()` or `physics: BouncingScrollPhysics()` is common.
  - Snapping can be achieved via `PageController` if using `PageView`, but `ListView` is easier for cards with custom widths.

## 📝 ADR (Architectural Decision Record)

No significant architecture change needed, just a UI refactor.
