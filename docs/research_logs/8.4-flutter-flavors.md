# Research Log - Story 8.4: Flutter Flavors Setup (Environment Separation)

## Overview

The objective is to establish separate environments for Development and Production to prevent experimental features from affecting real user data. This involves configuring Android Gradle flavors, separating Firebase configurations, and creating distinct entry points for the Flutter app.

## Current State

- **Android Gradle:** `android/app/build.gradle.kts` uses a single `applicationId`: `com.taqiyyaghazi.uangku`.
- **Firebase:** A single `google-services.json` exists in `android/app/`. There is no `firebase_options.dart` in `lib/`.
- **Entry Point:** `lib/main.dart` initializes Firebase using default options.

## Proposed Implementation

### 1. Android Flavor Setup

Modify `android/app/build.gradle.kts` to include:

- `flavorDimensions += "app"`
- `dev` flavor: `applicationIdSuffix = ".dev"`, `app_name: "Uangku Dev"`
- `prod` flavor: `app_name: "Uangku"`

### 2. Firebase Configuration Separation

- Create `android/app/src/dev/google-services.json` (for Dev Project).
- Create `android/app/src/prod/google-services.json` (for Prod Project).
- Remove the root `android/app/google-services.json`.

### 3. Entry Points & AppConfig

Create `lib/core/config/app_config.dart` to manage environment state.

Create two main entry points:

- `lib/main_dev.dart`: Sets environment to `dev`, initializes Firebase.
- `lib/main_prod.dart`: Sets environment to `prod`, initializes Firebase.

### 4. Firebase Options (Manual or Generated)

Since `firebase_options.dart` is missing and the project is using native configuration files, we have two choices:

1.  **Strict Native:** Let the native build system handle `google-services.json` based on the flavor folder. This is standard for Flutter and doesn't strictly _require_ `FirebaseOptions` in Dart if we use `Firebase.initializeApp()`.
2.  **Explicit Options:** Use `Firebase.initializeApp(options: ...)` to be more explicit and cross-platform friendly.

The AC #3 states: "masing-masing harus menginisialisasi Firebase dengan opsi (`FirebaseOptions`) yang sesuai dengan lingkungannya." This suggests Choice 2.

We will create `lib/firebase_options_dev.dart` and `lib/firebase_options_prod.dart`.

## Risks & Mitigations

- **Build Failures:** Flavor configuration in Gradle can be finicky. Mitigation: Use standard `productFlavors` and verify with `./gradlew tasks`.
- **Firebase Initialization Errors:** If options don't match the native configuration. Mitigation: Ensure `FirebaseOptions` match the `google-services.json` files.
