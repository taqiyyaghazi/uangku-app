# Research Log - Story 8.2.1 Cloud Sync

## Context

Goal: Sync budget limits and investment snapshots to Cloud Firestore.

## Findings

### 1. Investment Snapshots

- **Table:** `InvestmentSnapshots` in `lib/data/tables/investment_snapshots_table.dart`.
- **Repository:** `DriftInvestmentRepository` in `lib/data/daos/drift_investment_repository.dart`.
- **Sync Status:** Not yet hooked into cloud sync.

### 2. Budgets

- **Current State:** The app only has a global `monthly_budget` stored in the `AppSettings` table (Key-Value).
- **Mismatch:** User Story 8.2.1 AC #1 explicitly references "budget bulanan untuk sebuah kategori" and a Firestore path `users/{userId}/budgets/{categoryId}`.
- **Missing Infrastructure:** There is no `budgets` table or per-category budget management UI yet.

### 3. Sync Infrastructure

- **SyncRepository:** Already has a good pattern for Transactions, Categories, and Wallets.
- **FirestoreMapper:** Ready to be extended with new mapping logic.
- **SyncService:** Ready for new upsert/fetch methods.

## Proposed Resolution

To satisfy the User Story requirements:

1.  **Introduce `Budgets` table** in SQLite to support per-category limits.
2.  **Map SQLite Budgets** to Firestore `users/{uid}/budgets/{categoryId}`.
3.  **Map SQLite InvestmentSnapshots** to Firestore `users/{uid}/investments/{snapshotId}`.
4.  Expand `SyncRepository` and `SyncService` to handle these new units.
